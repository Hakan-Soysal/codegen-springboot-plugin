### Ext — `extFields` (referans §1 `:1142-1177`)
`op.ext()` varsa **TÜM** ns'ler için (trigger dahil — `.NET ExtPartial` birebir, genel prelude'a ns
ayrımı yapmaz) prelude yorumu (`// @{ns}.{name}(key=value, ...)`, `JsonNode` unquoted display) +
`report.realized("@{ns}.{name}", opId)` + `report.policy("{ns}-realization", "annotation/interceptor
(generator-policy)")`. Tanınan ns'ler ek sabit üretir: `@audit.*`(`category`)→`AUDIT_CATEGORY`,
`@metric.*`(`name`)→`METRIC_NAME`, `@http.*`(`route`/`method`/`query`/`header`)→`HTTP_ROUTE`/
`HTTP_METHOD`/`HTTP_QUERY`/`HTTP_HEADER`. **`@http` varsa AYRICA**
`policy("http-binding", "route/query/header detail (generator-policy)")` (.NET
`DotnetEmitter.cs:1160` `ExtPartial` paritesi; fixture `WriteAuditLog` `@http.endpoint` canlı örneği).

**Java-özgü tip güvenliği kararı (`jsonStrLiteral`):** .NET `JsonStr` bir JSON değerini (string/
number/bool) tipine göre C# literaline çevirir — `public const string X = 20;` gibi bir atama C#'ta
da tip uyuşmazlığı yaratır (latent varsayım: bu alanlar hep string). Java'da `String` alana sayısal/
boolean literal atamak KESİN derleme hatası olduğundan, `jsonStrLiteral` HER JSON değerini
(`JsonNode.asText()` ile) tırnaklı Java string literaline çevirir — üretilen kod her koşulda derlenir;
davranışsal fark yalnız (hiçbir fixture'da tetiklenmeyen) sayısal/boolean `category`/`name`/`route`
argümanında ortaya çıkar.

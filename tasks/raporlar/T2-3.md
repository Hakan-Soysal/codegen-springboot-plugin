## Yapılanlar (Section 5 adımları)

### 5.1 — BuildReport
`gen-core/src/main/java/techgen/core/report/BuildReport.java` (YENİ):
- `enum ConstructStatus { REALIZED, UNSUPPORTED, EMIT_CONFLICT, SILENT_DROP }`
- `record BuildEntry(String construct, String id, ConstructStatus status, String reason)`
- API: `realized`, `unsupported`, `conflict`, `silentDrop` (sabit reason
  `"manifest'te var; ne emit ne rapor (INV-7)"`), `policy(name,decision)` → `TreeMap<String,String>`
  (ordinal — HashMap KULLANILMADI).
- `covers(construct, owner)`: construct case-insensitive eşit ∧ `status != SILENT_DROP` ∧
  `idMatches`.
- `idMatches(id, owner)`: tam eşit VEYA (`startsWith` ∧ `id.length()>owner.length()` ∧ sınır-ayracı
  `{'#','-',':',' '}`) — `startsWith` ile BİTMEDİ, ayraç seti şart koşuldu.
- `clean()`, `silentDrops()` — C# ile birebir.
- `toJson()`: entry'ler `(construct,id)` ordinal (`String::compareTo`) sıralı; kök
  `{"constructs":[...],"policies":{}}`; her entry `LinkedHashMap` ile açık alan sırasıyla
  (`construct,id,status,reason`) yazıldı (Jackson POJO default alan-sırası KULLANILMADI); `reason`
  null ise `Map`'e hiç `put` edilmedi (alan yazılmaz); `DefaultPrettyPrinter` + `DefaultIndenter("  ",
  "\n")` ile 2-space/LF pretty-print. `writeTo(Path)` sonuna `"\n"` eklendi.

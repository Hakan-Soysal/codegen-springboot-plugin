## Notlar / kararlar

- Few-shot Java kod örnekleri (Command/Query/saga/idempotent/pagination/Trigger/Subscription/Boundary/
  test-arrange), gerçek `SpringEmitter.java` çıktısındaki sınıf/metot adlarıyla (ör. `{Op}HandlerBase`,
  `{Op}Guards.validation{N}`/`rule{N}`, `{Entity}Invariants.invariant{N}`, `THROWABLE_ERRORS`,
  `IDEMPOTENCY_KEYS`, `PAGINATION_STRATEGY`, marker metinleri `"{opId}: iş mantığı doldurulacak"` /
  `"{Ext}.{op}: doldurulacak"` / `"{opId}{T}Trigger.start: doldurulacak"` / `"{cls}.handle:
  doldurulacak"`) grep ile teyit edilip birebir kullanıldı — icat edilmiş isim yok.
- `archetype-playbooks.md`'ye 9. arketip olarak **test-arrange** eklendi. Bu, dotnet kardeşin
  `archetype-playbooks.md`'sinde YOKTU (o dosyada 8 arketip var), ancak SKILL.md (T9.1, zaten commit'li)
  Faz 2 tablosunda `test-arrange`'i ayrı bir arketip olarak listeliyor ve Faz 4'te ayrı bir alt-bölüm
  ("Faz 4 — test-arrange doldurma kuralı") ayırıyor; `conformance-testler-skill-sozlesmesi.md` §C.5
  eval#9 de test-arrange senaryosunu içeriyor. Playbook dosyasına bu arketipin few-shot'ını eklemek —
  SKILL.md'nin zaten attıfladığı içeriği tamamlamak — kapsam sapması değil, tutarlılık gereğidir; PM'e
  bilgi notu olarak burada belirtiliyor (sessiz sapma yok kuralına uyum).
- `evals.json`'da senaryo 4/5'teki `Order.userId`/`Order.customerId` örnek alanları camelCase Java
  field adlarına çevrildi (dotnet kardeşte PascalCase `Order.UserId`); anlam/aday-sayısı aynı kaldı.
- `CancellationToken` .NET'e özgü olup Java handler imzasında (`capability.json
  handlerSurfaceMap.operation`) karşılığı yok; tüm few-shot'larda ve verify-loop/gap-protocol
  metinlerinde kaldırıldı (sessiz atlama değil — T9.1'in zaten kilitlediği imza ile tutarlılık).

### Pagination (referans §1 `:1432-1446` {Op}.Page.g.cs + §7)

- `{Op}HandlerBase`'e yeni bölüm (`paginationFields`): `PAGINATION_STRATEGY = "{strategy}"` +
  (`pagination.size()!=null` ise) `DEFAULT_PAGE_SIZE = {size}`; `report.realized("pagination", opId)`
  + `report.policy("pagination-strategy", "{strategy} (generator-policy)")` +
  `report.policy("cursor-token", "opaque (generator-policy)")`.
- Dönüş tipi tüm katmanlarda (`execute` abstract/override, throws fabrikaları, Endpoint) `Result<
  Page<{Ret}>>` — `retType` hesaplaması `emit()` içinde tek noktada sarmalanıyor (`.NET ReturnType(op)`
  paritesi).
- İstek record'una (`{Op}Command`/`{Op}Query`) opak cursor (veya `strategy=="offset"` ise `Integer
  offset`) + `int size` alanı eklenir (`.NET RequestFields` paritesi) + üstüne pagination açıklama
  yorumu (`strategy`, `size`, `ORDER BY {alan} {yön}, ...`).
- GET Endpoint route'unda (yalnız body-bind OLMAYAN servingler) `@RequestParam(required = false)
  String cursor` (veya `Integer offset`) + `@RequestParam(defaultValue = "{size|20}") int size` eklenir
  ve request kurucusuna geçirilir (`.NET MapLine` paritesi; Java idiom: Spring `defaultValue` — .NET'te
  minimal-API lambda param default'u yok, en yakın idiomatik eşleme).

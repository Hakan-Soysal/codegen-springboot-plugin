`"NotProcessable"`/`"Success"` string literal'i içermez — beklenen değerler yalnız
`spec.assert_()`'ten okunur (A3).

'Success' substring'i yok" testi geçmiyor. Task §5.3 metodun adını `tryGetSuccessFieldDecimal`
olarak zorunlu kıldığından, adı değiştirmek de bir spec sapması olurdu — sessizce seçim

### 5.2 — GmBuilder
`gen-core/src/main/java/techgen/core/pipeline/GmBuilder.java` — akış CoreTemplate1 ile birebir:
1. `linked = "linked".equals(m.mode())`.
2. JoinError #1: `linked && m.contract()!=null && contract==null` →
   `"linked mod ama operations.json çözülemedi: " + m.contract()`.
3. Linked'de `contractOps`/`contractEntityIds` Id-index'lenir; standalone'da boş kalır.
4. Operasyonlar `OperationJson::id` ile ordinal sıralanır (Java `String.compareTo`/doğal sıralama
   — Collator KULLANILMADI), her biri `buildOp` ile işlenir: JoinError #2 (op realizes) +
   `isCommand = !creates.isEmpty() || !updates.isEmpty() || !deletes.isEmpty()` (contract `kind`
   kullanılmadı — yalnız manifest access 4-anahtarı).
5. Entity realizes denetimi (JoinError #3): linked'de her `rid` `contractEntityIds`'te olmalı.
6. Tüm koleksiyonlar sözleşme anahtarlarıyla sıralı: Modules/Deployables/Externals/Uncharted=name;
   Entities/Types/Events/Errors=id; Subscriptions=(event.module,event.name,consumer.op);
   CallEdges=(from,to.system,to.op).
7. `TypeEnv` kuruldu: `opParams`(opId→paramName→type), `entityFields`(entityId→fieldName→type).
8. `testPlan = TestPlan.empty()` (bkz. yukarıdaki karar).

### 5.4 — ExprWalk (dil-nötr yürüyüş)
`gen-core/src/main/java/techgen/core/predicate/ExprWalk.java`:
- `Render<T>` callback arayüzü: `binary(nodeKind, op, left, right)` (op yalnız cmp/arith'te dolu;
  implementasyon SONUCU HER ZAMAN parantezli üretmeli — sözleşme javadoc'ta açık), `path(path,
  propName)`, `literal(litKind, value, decimalContext)`. **Hiçbir Java kod parçası ExprWalk
  içinde string-concat ile üretilmiyor** — render tamamen callback'e bırakıldı (T3.5 bunu Java'ya
  bağlayacak).
- `AggNode/CallNode/DurationNode` → `UnsupportedConstruct("unsupported expr node: <TipAdı>")`.
- Op'suz cmp/arith (nodeKind∈{cmp,add,sub,mul,div} && op==null) → `UnsupportedConstruct`; and/or
  hiçbir zaman op taşımadığından bu kural onlara UYGULANMAZ (sözleşmeyle uyumlu, testle kanıtlı).
- `typeOf(node, resolveType)`: Decimal>Double>Int baskınlığı (add/sub/mul/div), literal tam-sayı
  double→"Int".
- `inferLiteralTypes(root)`: karşılaştırma bağlamından path tip-ipucu (ilk eşleşme kazanır —
  `putIfAbsent`).
- Distinct path toplama: görülme sırasıyla, `propName` (camel-join) anahtarıyla dedup.
- `propName(path)`: **camel-join** (task metninin PascalJoin yerine açıkça istediği biçim) —
  ilk segment küçük harfle başlar, sonrakiler Pascal-case: `["resource","creditLimit"]` →
  `"resourceCreditLimit"`.

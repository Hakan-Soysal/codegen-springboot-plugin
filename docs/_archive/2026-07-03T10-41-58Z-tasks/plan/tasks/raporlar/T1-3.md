## Yapılanlar

### `techgen.core.pipeline.Loader`
`gen-core/src/main/java/techgen/core/pipeline/Loader.java` — CoreTemplate1
`Pipeline/Loader.cs` (READ-ONLY okundu) ile birebir davranış:

- `loadManifest(Path path)`:
  - `Files.exists(path)` false → `LoadError("manifest bulunamadı: " + path)`.
  - Var ama parse hatası (veya IO hatası) → `LoadError("manifest ayrıştırılamadı: " + e.getMessage())`.
    `Files.readString` + `Json.parse` tek `try` içinde; `JsonProcessingException` `IOException`'ı
    extend ettiği için tek `catch (IOException e)` bloğu her iki durumu da kapsıyor (C#'taki
    `JsonException`-özel yakalamanın Java eşdeğeri — okuma hatası da pratikte "ayrıştırılamadı"
    olarak raporlanıyor, davranış sözleşmesinde bu ayrım test edilmiyor).
  - Fatal: her iki dal da `LoadError` (unchecked `RuntimeException`) fırlatıyor — çağıran yutmuyor.
- `loadContract(Path manifestPath, String contractPath)`:
  - `contractPath == null` → `null` (standalone mod).
  - Yol `manifestPath.getParent()`'e göreli çözülür (`dir.resolve(contractPath).normalize()`);
    parent `null` ise (bare filename) `Path.of(".")` fallback.
  - Dosya yok VEYA parse hatası → `null` — **throw YOK**. Hata join aşamasına (T2.1 GmBuilder)
    devrediliyor (sözleşme §4/§8).

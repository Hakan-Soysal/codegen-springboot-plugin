## 5. Changes

### Step 5.1 — EmitWriter
**File:** `gen-spring/src/main/java/techgen/spring/EmitWriter.java`
**Action:** Add — alanlar: `Path outDir`, `List<WrittenFile> written` (path+sha256). Metotlar:
- `writeAlways(String relPath, String content)`: relPath `.java` ile bitiyorsa başına GenHeader:
  ```java
  // <auto-generated/>  — techgen-spring; elle DÜZENLEME: her üretimde ezilir
  ```
  (tek satır + `\n`). Dizinleri oluştur. **write-only-if-changed**: mevcut içerik byte-aynıysa dosyaya
  DOKUNMA (mtime korunur). Her durumda `written`'a (relPath, sha256(finalContent)) ekle.
- `writeIfAbsent(String relPath, String content)`: dosya VARSA hiçbir şey yapma; yoksa dizin oluştur +
  yaz. `written`'a ASLA ekleme.
- `migrateSeamIfFlat(Path oldFlat, Path newSlice)`: eski yol varsa ve yeni yol yoksa → `Files.move`
  (kopya değil), sonra normal writeIfAbsent akışı.
- Tüm içerikler `\n` satır sonlu; `Files.writeString(..., UTF_8)`.

### Step 5.2 — Prune + provenance yazımı
**File:** aynı sınıf, `finishAndPrune()` metodu
**Action:** Add — sıra birebir:
1. Önceki `provenance.json`'ı `ProvenanceIo.tryRead` ile oku; null (yok/bozuk) → **prune atlanır**.
2. Önceki `Generated` path'lerinden bu run'da `written`'da OLMAYANLAR → dosyayı sil (orphan prune).
   `written`-dışı hiçbir path'e dokunulmaz (human dosyaları provenance'ta zaten yok).
3. Silme olduysa: artık-boş dizinleri en-derin-önce sil.
4. Yeni provenance'ı yaz: generator="techgen-spring", version=artefakt sürümü, files=`written`
   ordinal path-sıralı, atomik (temp+move), sonda `\n`.

### Step 5.3 — Testler
**File:** `gen-spring/src/test/java/techgen/spring/EmitWriterTest.java`
**Action:** Add (hepsi `@TempDir` ile):
- writeAlways: `.java` GenHeader alıyor, `.xml`/`pom.xml` ALMIYOR.
- write-only-if-changed: aynı içerik ikinci yazımda mtime değişmiyor (mtime'ı 5sn geriye set edip doğrula).
- writeIfAbsent: mevcut dosya (farklı içerikli) EZİLMİYOR; yok ise oluşuyor.
- prune: run1'de A+B yaz → finish; run2'de yalnız A yaz → finish → B silinmiş, boş dizini silinmiş;
  human dosyası (writeIfAbsent ile yazılmış C) DURUYOR.
- bozuk provenance.json → prune atlanıyor (B duruyor), yeni provenance yazılıyor.
- migrateSeamIfFlat: düz yoldaki dosya taşınıyor (eski yol yok, yeni yol var, içerik aynı).
- provenance dosyası: ordinal sıralı, lowercase-hex sha256, sonda `\n`.

## 8. Anti-patterns
- DO NOT prune'u "gen/ altındaki her şeyi sil, yeniden yaz" diye gerçekle — YALNIZ önceki provenance
  listesi − bu run yazılanları; aksi insan dosyası riski + mtime fırtınası.
- DO NOT GenHeader'ı writeIfAbsent çıktısına ekle — header yalnız üreteç-sahibi `.java`.
- DO NOT `File.delete` başarısızlığını yut — sürpriz durum → exception (sessizlik yasak).
- DO NOT sha256'yı dosyadan geri okuyarak hesapla — içerik string'inden (yazım atlansa bile).
